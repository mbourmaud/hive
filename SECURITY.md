# Security Policy

## Supported Versions

We release patches for security vulnerabilities in the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 3.x.x   | :white_check_mark: |
| < 3.0   | :x:                |

## Reporting Vulnerabilities

We take the security of Hive seriously. If you believe you have found a security vulnerability, please report it to us through GitHub Issues.

**Please do the following:**

1. Go to our [GitHub Issues page](https://github.com/mbourmaud/hive/issues)
2. Create a new issue with the label "security"
3. Provide a detailed description of the vulnerability, including:
   - Steps to reproduce the issue
   - Potential impact
   - Suggested fix (if any)

**Please do not:**

- Disclose the vulnerability publicly until it has been addressed
- Exploit the vulnerability beyond what is necessary to demonstrate it

## Security Best Practices

When using Hive, we recommend:

- Keep Hive updated to the latest version using `hive update`
- Review plans before executing them with drones
- Use appropriate access controls for your git repositories
- Ensure your Claude CLI credentials are properly secured
- Monitor drone activity logs regularly

## Response Timeline

- **Initial Response**: We aim to acknowledge security reports within 48 hours
- **Status Updates**: We will provide updates on the progress every 5-7 days
- **Resolution**: We will work to address confirmed vulnerabilities in a timely manner, with severity determining priority

## Security Updates

Security updates will be released as patch versions and announced through:

- GitHub Releases
- Release notes in the repository

Thank you for helping keep Hive and its users safe.
