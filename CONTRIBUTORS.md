## How to Contribute

We welcome contributions to Hive! Here's how you can help:

### Getting Started

1. **Fork the repository** and clone it locally
2. **Install dependencies**: Ensure you have Rust, git, gh CLI, and claude CLI installed
3. **Initialize Hive**: Run `hive init` in your local repository
4. **Create a branch**: Use a descriptive name for your feature or fix

### Development Workflow

1. **Make your changes**: Follow the existing code style and conventions
2. **Test your changes**: Run the test suite to ensure nothing breaks
3. **Commit your work**: Write clear, descriptive commit messages
4. **Submit a pull request**: Describe what you've changed and why

### Guidelines

- **Code Quality**: Write clean, maintainable Rust code
- **Documentation**: Update relevant documentation for any new features
- **Testing**: Add tests for new functionality
- **Issues First**: For major changes, open an issue to discuss before implementing

### Areas for Contribution

- Bug fixes and improvements
- New features and enhancements
- Documentation updates
- Test coverage improvements
- Performance optimizations

### Questions?

Feel free to open an issue for questions or discussions about contributing to Hive.
